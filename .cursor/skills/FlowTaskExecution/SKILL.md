# Flow выполнения задачи

Единый порядок выполнения любой задачи агента. **Применять к каждой задаче.**

## 1. Анализ

- Разобрать указанные пользователем компоненты, файлы, сценарии.
- Найти связанные компоненты (зависимости, вызовы, события, контроллеры, View).
- При необходимости использовать навыки проекта (template-architecture, template-mvc-views, template-eventmanager, template-winter-di, template-save-config и т.д.) и Unity MCP (ресурсы/инструменты) для понимания контекста.

## 2. Составление TODO плана

- Сформировать список конкретных шагов (TODO) для выполнения задачи.
- Использовать инструмент TodoWrite: создать задачи с ясными формулировками, при необходимости разбить на подпункты.

## 3. Выполнение по TODO

- Выполнять пункты плана по порядку.
- **После каждого выполненного TODO** проверять ошибки через Unity MCP:
  - `read_console` (фильтр по Error) — ошибки компиляции и рантайма;
  - при изменении скриптов — `execute_menu_item` с `menu_path: "Assets/Refresh"`, затем дождаться окончания компиляции (ресурс `editor_state`, поле `isCompiling`).
- Отмечать завершённые пункты в TodoWrite.

## 4. Проверка задачи

- Выполнить проверку по навыку **TaskVerification** (изначальные и дополнительные требования, код-стайл).
- Задействовать Unity MCP: консоль, компиляция, при необходимости сцены/ассеты/GameObject по контексту задачи.
- Если проверка пройдена — выполнить Unity Refresh (если ещё не выполнялся) и завершить задачу.

## 5. Цикл при ошибках или нарушениях

- Если **задача выполнена некорректно** (не сделано то, что просил пользователь, пропущены пункты),
- ИЛИ **есть ошибки** (консоль, компиляция),
- ИЛИ **есть нарушения код-стайла** из Rules (user rules, workspace rules),

то **вернуться к пункту 2**: пересмотреть и при необходимости скорректировать TODO план, затем снова выполнить пункты 3 и 4. Повторять, пока все TODO не будут выполнены и проверка (TaskVerification + Unity MCP) не пройдёт без замечаний.

## Краткая последовательность

1. Анализ → 2. TODO план → 3. Выполнение (после каждого TODO — проверка через Unity MCP) → 4. TaskVerification + Unity MCP → 5. При ошибках/нарушениях → снова 2–4.
